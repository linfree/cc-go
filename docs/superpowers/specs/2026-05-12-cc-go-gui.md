# cc-go GUI 启动改造

**日期**: 2026-05-12
**状态**: 设计完成

## 目标

将 cc-go 从命令行启动方式改为双击 exe 直接运行，带托盘图标和状态窗口，引导用户到 Web 管理面板。

## 用户场景

1. 用户双击 `cc-go.exe` → 看到状态窗口 + 托盘图标
2. 状态窗口显示服务运行状态，有"打开 Web 管理面板"按钮
3. 点击按钮 → 跳转浏览器打开 `http://localhost:18080`
4. 关闭窗口 → 最小化到托盘，服务继续运行
5. 右键托盘图标 → 菜单：打开面板 / 显示窗口 / 退出
6. 点击退出 → 停止所有服务，程序退出

## 文件变更

### 新增: `internal/ui/`

```
internal/ui/
  ui.go          # Run() / Quit() 接口 + 平台分派
  ui_windows.go  //go:build windows — 托盘 + WebView 状态窗口
  ui_other.go    //go:build !windows — 托盘 + 自动打开浏览器
```

**ui.go** — 公共接口：
```go
package ui

type UI interface {
    Run(onReady func())
    Quit()
}

func New(port int, logo []byte) UI { /* 平台分派 */ }
```

**ui_windows.go** — Windows 实现：
- `github.com/getlantern/systray` 托盘图标
- `github.com/jchv/go-webview2` 状态窗口（纯 Go，无 CGO）
  - 标题 "cc-go"，尺寸 480x320
  - 加载 `http://localhost:{port}/status`
  - 关闭时隐藏而不是退出
- 托盘菜单：打开 Web 面板 / 显示窗口 / 分隔线 / 退出
- 退出前调用 `bridge.StopSession()` + `wechat.Stop()`

**ui_other.go** — Linux/Mac 实现：
- `github.com/getlantern/systray` 托盘图标
- 启动后自动打开浏览器
- 托盘菜单：打开 Web 面板 / 退出

### 新增: `/status` 路由

在 embed.go 旁边新增一个内联状态页路由，返回极简 HTML（不依赖 React 构建），包含：
- 绿色圆点 + "服务运行中"
- "打开 Web 管理面板" 按钮（`window.open` 外部浏览器）
- 显示 Web 端口号

### 修改: `cmd/cc-go/main.go`

- 去掉 cobra CLI 依赖
- 改为：加载配置 → 初始化 store/bridge/wechat/server → 创建 UI → `ui.Run(onReady)`
- `onReady` 中启动 HTTP 服务
- 保留所有核心逻辑（信号处理等）

### 修改: `cmd/cc-go/embed.go`

新增 `/status` 路由注册。

### 新增依赖

```
github.com/getlantern/systray   # 跨平台托盘
github.com/jchv/go-webview2     # Windows WebView2 窗口（纯 Go，无 CGO）
```

### 修改: `Makefile`

- `go-webview2` 是纯 Go，无 CGO 需求，保持现有构建方式

## 跨平台策略

| 平台 | 窗口 | 托盘 | 浏览器 |
|------|------|------|--------|
| Windows | WebView2 状态窗口 | ✓ | 按钮/菜单跳转 | pure Go |
| Linux | 无 | ✓ | 启动时自动打开 | pure Go |
| macOS | 无 | ✓ | 启动时自动打开 | pure Go |

go-webview2 通过 Win32 COM API 直接调用 WebView2，无需 CGO。Linux/Mac 退化为纯托盘+浏览器模式。

## 状态窗口 HTML（`/status`）

内联样式极简页面，不依赖前端构建：
- 绿色脉冲圆点 + "cc-go 运行中"
- 端口号显示
- "打开 Web 管理面板" 按钮（大号主色按钮）
- 按钮点击通过 `window.open(href)` 在外部浏览器打开

## 构建

```bash
# 全平台统一构建（纯 Go，无 CGO）
go build -o cc-go ./cmd/cc-go/

# Windows
go build -o cc-go.exe ./cmd/cc-go/
```
