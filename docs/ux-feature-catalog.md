# cc-go 产品功能清单与 UX 优化建议

> 供 UI 设计师参考，用于设计效果图。

---

## 1. 产品定位

cc-go 是一个通过微信机器人远程管理 Claude Code 的工具。用户在电脑上运行 cc-go，通过微信或 Web 界面与 Claude Code 交互——启动/接管会话、查看输出、审批权限请求。

**核心用户场景：**
- 移动办公时，通过微信操控电脑上的 Claude Code
- 在 Web 界面上查看会话历史、审批权限、管理配置
- 长时间运行的任务，通过手机远程监控进度

## 2. 当前页面与功能

### 2.1 全局布局（MainLayout）

- 左侧固定侧边栏（200px），包含：
  - Logo + 应用名称 "cc-go"
  - 5 个导航项：会话日志、会话管理、微信连接、Claude 配置、系统设置
  - 底部：当前活跃会话指示器（绿色 Tag + 会话名称 + 工作目录）

### 2.2 微信连接页（WechatBind）

| 功能 | 状态 |
|------|------|
| 显示连接状态（已连接/未连接） | ✅ 已实现 |
| 生成二维码供微信扫码绑定 | ✅ 已实现 |
| 断开连接按钮 | ✅ 已实现 |
| 每 5 秒自动轮询状态 | ✅ 已实现 |

**缺失体验：**
- 无断开确认弹窗（点击即断开）
- 无二维码过期提示/倒计时
- 无扫描成功动画反馈
- 不显示已绑定的微信账号信息
- 不显示登录时间或会话到期时间（微信会话 24 小时过期）
- 不处理"即将过期"状态的提醒

### 2.3 会话管理页（SessionList）

| 功能 | 状态 |
|------|------|
| 列表展示所有会话（名称、工作目录、消息数、最后活跃、状态） | ✅ 已实现 |
| 接管（resume）已停止的会话 | ✅ 已实现 |
| 停止（stop）活跃会话 | ✅ 已实现 |
| 删除会话（带确认弹窗） | ✅ 已实现 |
| 新建会话弹窗（工作目录、名称、模型） | ✅ 已实现 |
| 状态 Tag 颜色区分（active=绿, idle=蓝, stopped=灰, error=红） | ✅ 已实现 |
| 每 10 秒自动刷新 | ✅ 已实现 |

**缺失体验：**
- 无搜索/筛选功能（按名称、状态、工作目录筛选）
- 不显示 git 分支信息（后端有数据但未展示）
- 不显示会话使用的模型
- 新建会话表单的模型字段是自由文本而非下拉选择
- 操作按钮无加载中状态
- 删除按钮只有图标无文字，含义模糊
- 无批量操作（多选、批量停止/删除）
- 新建会话的工作目录需要手动输入路径，无法通过文件浏览器选择

### 2.4 会话记录页（SessionChat）

| 功能 | 状态 |
|------|------|
| 聊天式展示会话历史（用户/助手气泡） | ✅ 已实现 |
| 系统消息居中显示（附件、系统事件、队列操作等） | ✅ 已实现 |
- 思考过程可折叠展示 | ✅ 已实现 |
| 工具调用可折叠展示（tool_use + tool_result 合并） | ✅ 已实现 |
| 工具执行结果绿色卡片显示 | ✅ 已实现 |
| 活跃会话每 3 秒轮询更新 | ✅ 已实现（仅消息数变化时才更新） |
| 首次加载自动滚到底部 | ✅ 已实现 |
| 24 小时制时间格式 | ✅ 已实现 |

**缺失体验：**
- 无面包屑/返回导航（只能通过侧边栏回列表）
- 不显示当前会话的元信息（名称、状态、工作目录、模型）
- 无 Markdown 渲染——助手回复是纯文本（pre-wrap），无代码高亮
- 无复制按钮
- 无发送消息输入框（只能查看，不能从 Web 发消息）
- 无导出/下载会话记录功能
- 长历史无分页（一次加载全部）
- 硬编码 `maxHeight: 70vh`，不适配不同屏幕

### 2.5 会话日志页（LogViewer）

| 功能 | 状态 |
|------|------|
| 终端风格实时日志（深色背景、彩色标签） | ✅ 已实现 |
| 每种事件类型不同颜色（输入蓝、输出绿、工具橙、权限黄等） | ✅ 已实现 |
| 每 2 秒自动刷新 | ✅ 已实现 |

**缺失体验：**
- 无日志级别筛选（只看错误、只看工具调用等）

- 不显示时间戳
- 无暂停/恢复自动滚动切换（查看旧日志时会被新日志拉走）
- 无复制/导出功能
- 无清空按钮
- 只能看活跃会话的日志，无法查看历史会话日志
- 不显示完成事件的耗时和轮次信息

### 2.7 系统设置页（Settings）

| 功能 | 状态 |
|------|------|
| 基础设置表单（语言、端口、自动开浏览器、自动检测 CLI、权限模式） | ✅ 已实现 |
| 推送通知开关（权限请求、Claude 回复、工具调用、会话状态） | ✅ 已实现 |

- 三个功能区无视觉分隔（可改为 Tab 或分区卡片）

- 无恢复默认设置按钮

- 表单宽度固定 500px，宽屏浪费空间

## 3. 后端已实现但前端未利用的功能

| 功能 | 说明 | 建议 |
|------|------|------|
| WebSocket 实时推送 | 后端有完整的 WS 事件总线（会话状态变更、Claude 输出、权限请求、权限取消），前端完全没接入，全部用 HTTP 轮询替代 | 改用 WS 替代轮询，实现真正的实时体验 |
| 权限审批 Web UI | 后端能处理权限请求，但只能通过微信 /y /n 操作，Web 无入口 | 新增 Web 权限审批面板——实时接收权限请求弹窗，一键批准/拒绝 |
| git 分支信息 | 会话数据包含 git_branch 但前端不展示 | 在会话列表和详情页显示分支 Tag |
| 微信登录时间 & 到期提醒 | 后端返回 login_time 并有 24h 过期机制，前端不感知 | 显示登录时间、到期倒计时、即将过期预警 |
| 会话使用的模型 | 启动时可指定模型，但列表不展示 | 在会话详情页显示模型信息 |
| 完成事件耗时/轮次 | EventResult 包含 duration_ms 和 num_turns，日志不展示 | 在日志和会话详情中显示耗时和轮次统计 |
| 删除只删 DB 不删文件 | 删除会话后 JSONL 文件仍在，会话会重新出现 | 需修复：删除时同时清理文件 |

## 4. UX 优化建议（新增功能）

### 4.1 高优先级——核心体验提升

| 功能 | 说明 |
|------|------|
| **Web 权限审批面板** | 实时接收权限请求，显示工具名称和参数，一键批准/拒绝，支持批量操作。这是 Web 界面最大的功能缺口 |
| **Web 发送消息** | 在 SessionChat 页面添加输入框，可以直接从 Web 向 Claude 发消息，不再只依赖微信 |
| **实时推送（WebSocket）** | 替换所有 HTTP 载荷为 WebSocket，消除轮询延迟。状态变更、新消息、权限请求即时到达 |
| **Markdown 渲染** | 助手回复支持 Markdown（代码块语法高亮、列表、标题、链接），是 Claude 回复的核心展示需求 |

### 4.2 中优先级——专业感提升

| 功能 | 说明 |
|------|------|
| **会话详情头部** | 在 SessionChat 顶部显示会话名称、状态 Tag、工作目录、模型、创建时间、git 分支 |
| **面包屑导航** | 会话记录页显示「会话管理 > 会话详情」面包屑，方便返回 |
| **搜索和筛选** | 会话列表支持按名称/状态/工作目录搜索筛选 |
| **工作目录浏览器** | 新建会话时提供文件夹选择器，而非手动输入路径 |
| **微信会话到期提醒** | 显示登录时间、到期倒计时；即将过期时弹出提醒引导重新扫码 |
| **响应式布局** | 侧边栏可折叠，适配平板和窄屏 |

### 4.3 低优先级——锦上添花

| 功能 | 说明 |
|------|------|
| **复制消息按钮** | 每条消息旁提供复制按钮 |
| **导出会话记录** | 下载为 Markdown 或 JSON 文件 |
| **日志筛选** | 按事件类型筛选，暂停自动滚动 |
| **深色模式** | 全局主题切换，日志页天然适合深色 |
| **键盘快捷键** | 常用操作快捷键（Ctrl+Enter 发消息、Esc 关闭弹窗等） |
| **会话统计卡片** | 仪表板首页：活跃会话数、总消息数、今日使用时间等 |

## 5. 信息架构建议

建议将页面重组为更清晰的层级：

```
首页（仪表板）
├── 会话概览（活跃会话卡片 + 快捷操作）
├── 最近活动（最新几条事件）
├── 统计数据（消息数、使用时间等）

会话管理
├── 会话列表（搜索、筛选、排序）
├── 会话详情
│   ├── 会话元信息头部
│   ├── 聊天记录（Markdown 渲染 + 发送消息输入框）
│   ├── 权限审批面板（实时）
│   └── 操作按钮（停止、删除、返回）

微信连接
├── 连接状态 + 账号信息
├── 二维码（含到期倒计时）
├── 断开/重连操作

Claude 配置（合并到设置页的一个 Tab）

系统设置（Tab 切换）
├── Tab 1: 基础设置
├── Tab 2: 推送通知
├── Tab 3: 机器人指令
├── Tab 4: Claude CLI 配置
```

## 6. 设计重点提示

1. **移动端适配很重要**——用户在手机微信上使用的场景很常见，Web 界面也要能在手机浏览器上正常操作
2. **权限审批是核心差异化功能**——让 Web 用户能实时审批权限请求，是 cc-go 的关键价值，UI 上要突出
3. **实时感**——尽量用 WebSocket 替代轮询，让用户感觉所有变化都是即时发生的
4. **Claude 回复的展示质量**——Markdown 渲染和代码高亮直接影响用户对输出质量的主观感受
5. **微信会话到期**——24 小时过期是硬约束，UI 上必须清晰提示到期状态和续期操作
6. **侧边栏活跃会话指示器**——当前的底部小块设计不够醒目，建议改为更突出的状态栏或浮动卡片
